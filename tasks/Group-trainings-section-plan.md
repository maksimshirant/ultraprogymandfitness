# План разработки раздела "Групповые тренировки"

## Цель задачи

Разработать отдельный роут раздела **«Групповые тренировки»**, который:

* показывает список направлений
* объясняет различия между ними
* показывает тренера каждого направления
* помогает быстро принять решение
* содержит кнопку записи внутри каждой карточки
* интегрирован в существующий layout сайта

Основная задача страницы — привести пользователя к записи на тренировку.

---

# Общая структура страницы

Страница должна состоять из следующих блоков:

1. Hero секция
2. Быстрая навигация по направлениям
3. Группировка направлений по категориям
4. Карточки направлений
5. Блок помощи выбора направления
6. Финальный CTA блок

---

# 1. Hero секция

Содержимое:

* Заголовок: **Групповые тренировки**
* Подзаголовок (2–3 строки)
* Основная кнопка CTA
* Вторичная кнопка перехода к направлениям

Пример:

Заголовок:

Групповые тренировки для детей и взрослых

Подзаголовок:

Силовые, функциональные и восстановительные направления с тренерами клуба. Подберите тренировку под свои цели и уровень подготовки.

Кнопки:

* Записаться на тренировку
* Выбрать направление

---

# 2. Быстрая навигация по направлениям

Под hero размещается горизонтальный список кнопок:

* Единоборства
* ЛФК
* Ягодичный памп
* Единоборства (дети)
* Круговая силовая
* Функциональный тренинг
* Стретчинг
* АФК

Поведение:

Desktop:

* переход по якорям

Mobile:

* горизонтальный scroll

Назначение:

Позволяет быстро перейти к нужному направлению без прокрутки всей страницы.

---

# 3. Группировка направлений

Карточки должны быть разделены на смысловые группы.

## Активные тренировки

* Единоборства
* Ягодичный памп
* Круговая силовая
* Функциональный тренинг

## Восстановление и здоровье

* ЛФК
* АФК
* Стретчинг

## Детские тренировки

* Единоборства (дети)

Назначение:

Снижает когнитивную нагрузку пользователя.

---

# 4. Карточка направления

Карточки располагаются вертикальным списком.

Формат:

Desktop:

двухколоночная карточка

Mobile:

одна колонка

---

# Структура карточки

Карточка должна содержать:

1. бейдж направления
2. название направления
3. имя тренера
4. краткое описание
5. список преимуществ
6. кнопка записи
7. ссылка подробнее

---

# Пример структуры карточки

## Верхняя часть

Бейдж:

Групповое направление

Название:

Функциональный тренинг

Подпись:

---

## Краткое описание

Пример:

Комплексные упражнения для развития силы, координации и общей физической подготовки.

---

## Преимущества

Формат:

3 коротких пункта

Пример:

* развитие силы
* улучшение координации
* повышение выносливости

---

# Кнопка записи

Кнопка должна присутствовать внутри каждой карточки.

Текст кнопки:

Записаться на тренировку

Поведение:

Передаёт bookingAction направления.

---

# Кнопка подробнее

Ссылка раскрывает полный description направления.

Назначение:

Позволяет:

* сохранить компактность интерфейса
* не перегружать страницу текстом
* сохранить SEO

---

# Отображение тренера

Имя тренера должно быть выделено отдельным элементом.

Формат:

Тренер: Александр

Если есть фото:

используется мини аватар

Если фото нет:

используется текстовый блок

Кнопка:

Оставить заявку

---

# Рекомендуемая структура данных

Рекомендуется расширить структуру направления:

```
{
  key: string
  text: string
  trainer: string
  shortDescription: string
  benefits: string[]
  category: "active" | "recovery" | "kids"
  audience?: string
  badge?: string
  description: string
  action: string
  bookingAction: string
  confirmBookingAction: string
}
```

Назначение:

Позволяет:

* разделять карточки по категориям
* управлять UI без логики внутри компонентов
* переиспользовать данные

# Требования к UX

Карточка должна отвечать на вопросы пользователя:

Что это

Кому подходит

Кто ведёт

---

# Требования к UI

Карточка должна содержать:

* мягкий фон
* скругления
* внутренние отступы 24–32px
* вертикальный gap между элементами

Hover эффект:

лёгкий подъём карточки

---

# Ошибки которых нужно избежать

Нельзя:

использовать длинные сплошные тексты

делать одну кнопку записи на всю страницу

использовать таблицу вместо карточек

перегружать карточки иконками
